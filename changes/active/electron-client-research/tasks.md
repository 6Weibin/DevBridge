# DevBridge Electron PoC 任务清单

## 任务列表

- [x] 1. 创建独立 `DevBridge-Electron` 工程结构
  - 目标：新增 Electron 工程，不修改 `DevBridge-Front` 和 `DevBridge-Server` 源码。
  - 预期结果：存在可安装依赖、可运行脚本和清晰目录结构。

- [x] 2. 实现 Electron 客户端启动逻辑
  - 目标：启动静态前端服务，启动或复用本地后端，健康检查通过后加载页面。
  - 预期结果：主进程能管理服务生命周期和错误提示。

- [x] 3. 实现资源准备与打包配置
  - 目标：构建时收集前端 dist、后端 jar、tools，并尽量生成内置 Java runtime。
  - 预期结果：`npm run package` 能生成客户端包所需资源。

- [x] 4. 安装依赖并完成客户端打包
  - 目标：完成 Electron 工程依赖安装、资源准备和桌面客户端包输出。
  - 预期结果：产出可测试的客户端包路径。

- [x] 5. 验证与交付说明
  - 目标：执行可行的构建/检查命令，记录输出包和限制。
  - 预期结果：用户可按说明直接测试客户端包。

- [x] 6. 调整 Electron 专用端口
  - 目标：Electron 启动的前端服务使用 `15173`，后端服务使用 `18180`，不占用 `5173` 和 `8080`。
  - 预期结果：客户端仍可加载 `DevBridge-Front` 页面，并将页面中的 `8080/api/*` 请求转发到 Electron 后端 `18180`。
