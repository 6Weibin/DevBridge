### Why（意图）

当前 AI 日志分析仍偏“单业务接口”，扩展设备诊断和操作时会不断新增专用接口。需要把 ADB CLI 全能力沉淀为 MCP 工具，让 AI 通过工具调用完成诊断和设备管理编排。

### What（范围）

#### 做什么

- [ ] 建立覆盖 ADB 顶层命令全能力的 MCP 工具集，基准为项目内置 ADB `1.0.41 / 37.0.0-14910828`。
- [ ] 支持任意 `adb shell COMMAND...` 调用，但删除、卸载、重启、root、remount、sideload 等敏感操作必须对话确认。
- [ ] AI 对话和日志分析改为通过 MCP 工具获取上下文和执行动作。

#### 不做什么

- [ ] 不在本阶段做 iOS/Harmony MCP 工具全集。
- [ ] 不把 MCP 工具逻辑写入前端主页面。

### 初步技术方向

在后端 AI 模块内新增 MCP/Tool 边界，优先按 ADB help 自动/半自动生成工具目录；高风险命令走确认令牌机制。

### 成功标准

- [ ] ADB help 中的顶层命令均有 MCP 工具或等价通用工具覆盖。
- [ ] shell 任意命令可调用，敏感命令未经用户确认不会执行。

⚠️ 待确认：这里的 MCP 是否要求实现标准 Model Context Protocol Server，还是先作为 Spring AI Tool Calling 的本地工具抽象。
