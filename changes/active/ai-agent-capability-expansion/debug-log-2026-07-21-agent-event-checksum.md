# Agent Event 校验失败排查记录

## 排查步骤

- [x] 核对确认续跑失败日志，定位到 `AgentEventStoreException`。
- [x] 检查任务事件文件，确认 JSON 可正常解析，但所有事件在对象重序列化后摘要不一致。
- [x] 直接对磁盘 `payload` JSON 计算 SHA-256，结果与已存摘要完全一致。
- [x] 检查任务结果轮询链路，确认终态事件补偿失败会阻塞已持久化结果返回。

## 根因定位链

1. 用户确认后进入自动续跑。
2. 续跑为恢复调用预算而遍历 Agent Event。
3. 旧逻辑先反序列化事件，再序列化对象计算摘要。
4. `Map.copyOf` 和时间序列化设置会使重建后的 JSON 字节与磁盘原文不同，造成假性“校验和不匹配”。
5. 续跑在 Provider 调用前中断；同时结果轮询也因事件补偿报错，前端长时间停留在“正在思考”。

## 修复方案

- 摘要改为校验文件中实际存储的 `payload` JSON，然后再转换为 `AgentEvent`。
- 该方式兼容既有事件文件，不需要改写用户数据，真实篡改仍会校验失败。
- 任务文件已进入终态时，终态事件补偿失败仅记录警告，不再阻断前端获取最终结果。
