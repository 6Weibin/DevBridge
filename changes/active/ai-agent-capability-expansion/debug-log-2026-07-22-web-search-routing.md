- [x] 检查 Electron 运行时 AI 与网络检索配置。
- [x] 检查天气问题的工具范围路由规则。
- [x] 检查产品 Prompt 对实时信息和配置缺失的约束。
- [x] 修复路由与 Prompt，并完成回归验证。

## 假设与验证

| 假设 | 验证结果 |
|------|----------|
| 旧系统提示词仍声明不支持联网 | 未命中；当前用户配置没有该文案 |
| 网络工具没有注册 | 未命中；运行时 MCP 已注册 66 个工具 |
| 天气问题没有进入网络工具范围 | 命中；路由词表未包含天气等实时公开信息 |
| Tavily 已配置但调用失败 | 未命中；运行时配置实际为未配置、未启用 |

## 根因

“今天成都天气怎么样”没有包含原路由规则要求的“联网、最新、官网”等词，因此被分配到设备管理工具范围。模型看不到 `web.search`，结合设备上下文错误地建议打开手机天气应用。同时，当前 Tavily 配置尚未填写，无法完成真实查询。

## 修复

- 将天气、空气质量、实时新闻、股价和汇率纳入网络检索意图。
- 产品 Prompt 明确实时公开信息优先调用 `web_search`。
- 网络工具未配置时要求模型提示进入 AI 配置完成设置，不再声称系统没有该能力。
- 使用原始天气问题补充精简路由回归断言。

## 补充修复：失败语义丢失

后续验证发现 Tool Gateway 会把 `WEB_SEARCH_NOT_CONFIGURED` 降级成通用 `TOOL_EXECUTION_FAILED`，Chat 兼容结果也没有把诊断详情交给模型，导致模型编造 Google/Bing 配置建议。

- Gateway 现在保留 `BusinessException` 的错误码、用户提示和脱敏详情。
- Chat 工具结果把诊断信息同步传给模型、SSE 和工具卡片。
- Prompt 和工具定义明确当前网络搜索只支持 Tavily。
- 端到端 SSE 验证返回 `WEB_SEARCH_NOT_CONFIGURED` 和 Tavily 配置步骤，不再出现 Google/Bing。

## 架构收敛：移除关键字路由

继续检查发现 `AiConversationService` 仍通过天气、官网、最新等关键字决定是否暴露网络工具，这与 `ToolCapabilityRouter`“不解析自然语言关键字”的设计原则冲突，也会遗漏未知表达。

- 删除网络关键字匹配逻辑和 `WEB_RESEARCH` 专用范围。
- 普通对话统一使用 `GENERAL_ASSISTANT`，同时向模型提供设备与网络工具。
- 模型通过 Tool Calling 自主判断是否需要 `web.search` 或 `web.fetch`。
- Local Shell 仍保持显式本机意图边界，不因网络工具调整而扩大。
- 使用“成都现在出门需要带伞吗？”完成真实验证；问题不含原关键字，模型仍自主调用 Tavily 并引用来源。
