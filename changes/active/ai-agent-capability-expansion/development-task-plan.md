## 1. 计划说明

本计划覆盖当前 AI 助手架构评估和二次自检发现的全部问题。实施原则如下：

- 保持 Spring Boot 模块化单体，不在当前阶段拆分微服务。
- 保留现有 Provider Gateway、ADB/Local Shell Planner、Risk、Confirmation、Executor 和 Audit 基础。
- 先完成后端 Agent 控制平面，再建设领域工具、RAG 和多 Agent。
- 前端只负责创建任务、订阅事件、提交确认和展示结果。
- 所有敏感操作、工具调用和跨步骤恢复必须由后端确定性控制。
- 单个任务工作量控制在 0.5 至 2 人天；工作量为初步预估，不包含外部 Provider 联调等待时间。

  -> M2 Tool Gateway / Router / 领域工具
