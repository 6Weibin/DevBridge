## 5. 遗留问题

M0 是规格与契约里程碑，不代表生产能力已经完成。以下问题仍未解决：

1. 当前前端仍承担旧链路 Agent 控制职责。
2. Task、Checkpoint、Event Store 和确认自动恢复尚无生产实现。
3. 工具输出、线程池、SSE 背压、进程树取消和资源锁风险仍存在。
4. 本地 API 认证、数据外发审批和不可信内容隔离尚未落地。
5. Tool Gateway、领域工具、Memory、RAG、多 Agent 和可观测性仍按 M1-M4 排期。

这些均已在 `development-task-plan.md` 中有明确后续任务，不属于 M0 验收失败。
