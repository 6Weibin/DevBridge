# AI 助手提示、流式中断与输入区遮挡排查记录

## 问题现象

- 顶部黄色提示“AI 执行失败，详情已显示在对话中。”出现后不会自动消失。
- AI 长回复或 ADB MCP 工具调用后出现“AI 流式响应中断：连接已关闭但未收到完成事件”。
- 默认欢迎语使用了独立气泡样式，与普通 AI 回复格式不一致。
- 底部输入区的背景区域过高，并且自身毛玻璃效果会干扰对话窗口的整体毛玻璃背景。

## 排查步骤

- [x] 检查 `AiChatPanel.tsx` 顶部提示状态的生命周期。
- [x] 检查前端 SSE 完成事件判断，确认错误来自“连接关闭但未收到 done”。
- [x] 检查 `AiConversationService.streamChat` 和 SSE `onTimeout` 配置。
- [x] 检查系统欢迎语渲染分支和输入区浮层样式。
- [x] 执行前端构建、后端单测、静态资源同步和服务重启验证。

## 假设与验证

| 假设 | 验证结果 |
| --- | --- |
| 前端误判正常结束为中断 | 不成立。前端只有收到 `done` 才标记完成，连接关闭且没有 `done` 时报告中断符合协议。 |
| Provider 从开始就没有返回内容 | 不成立。错误提示中已接收多个 SSE 事件，说明流式链路曾正常输出。 |
| 后端 SSE 生命周期过短导致正常长任务被截断 | 成立。`SseEmitter` 总超时时间原为 70 秒，ADB MCP 工具调用叠加模型最终分析容易超过该窗口。 |
| 输入区大面积模糊导致视觉冲突 | 成立。底部浮层和输入卡片都带模糊/高背景，叠加后削弱对话区域毛玻璃效果。 |

## 根因链

用户请求触发 ADB MCP 工具调用 → 工具事件和部分模型内容通过 SSE 正常发送 → ADB 输出较大或模型继续分析耗时超过 70 秒 → Spring `SseEmitter` 到达总生命周期超时并关闭连接 → 后端无法稳定再发送最终 `done` 事件 → 前端接收到连接关闭且没有 `done`，按协议展示“流式响应中断”。

该问题不是 Markdown 渲染或前端滚动导致，核心是后端 SSE 总生命周期不满足长工具调用场景。

## 修复方案

- 将 `AiConversationService.CHAT_STREAM_TIMEOUT_MILLIS` 调整为 300 秒，覆盖 ADB MCP 工具调用和长回复的正常耗时窗口。
- 增加后端单测，约束流式超时时间不能回退到过短配置。
- 为顶部提示增加 5 秒自动清理逻辑，详细错误仍保留在对话区。
- 将 `system` 欢迎消息复用普通 AI 回复渲染分支，去掉独立气泡观感。
- 压低底部输入区渐变承托高度，移除输入卡片自身模糊，保留真实高度 spacer，避免最新消息被输入区遮挡。

## 验证结果

- `npx -y yarn@1.22.22 build`：通过。
- `mvn -Dtest=AiConversationServiceTest test`：通过，3 个测试。
- `mvn -DskipTests resources:resources`：通过，前端静态资源已同步到后端。
- 重启 `devbridge-backend` 后访问 `http://127.0.0.1:8080/` 返回 `200`。
