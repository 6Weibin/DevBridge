### Why

DevBridge 当前已经具备 USB 设备识别、设备详情、文件管理、应用管理、Android/iOS 实时日志采集、日志过滤和导出能力。前端主界面集中在 `DevBridge-Front/src/app/App.tsx`，后端日志能力集中在 `LogController`、`LogStreamService`、`LogCaptureService`，现有功能已经形成稳定的用户工作流。

本次需求是在不破坏现有工作流的前提下，增加一个可浮动唤起的 AI 助手。用户可以通过对话配置 AI、发起日志采集分析、查看分析结论，并在后续扩展 DeepSeek、GLM、Qwen、ERNIE、OpenAI 等 Provider 时保持统一体验。

目标效果：AI 助手作为独立能力叠加在现有界面上，默认不改变设备列表、页签、日志采集、文件管理和应用管理行为；用户关闭 AI 助手后，当前主界面状态保持不变。

### How

#### 1. 正常流程

1. 用户进入 DevBridge 主界面。
2. 页面右下角展示 AI 圆形入口。
3. 用户点击 AI 入口。
4. 系统查询 AI 配置状态。
5. 如果 AI 已配置，系统打开右侧浮动侧边栏。
6. 用户在侧边栏进行普通对话，系统展示 AI 回复。
7. 用户选择当前设备并请求日志分析。
8. 系统读取当前设备上下文和日志上下文，执行脱敏和条数限制。
9. 系统将处理后的上下文提交给 AI Provider。
10. 侧边栏展示分析结论，主界面当前设备、页签、日志过滤、文件选择和应用列表不被重置。

#### 2. 关键场景

#### 场景：首次点击 AI 图标时未配置
- **Given** 后端在线，AI 配置状态为未配置
- **When** 用户点击右下角 AI 图标
- **Then** 系统打开 AI 配置弹窗，并展示 API URL、API Key、模型、Provider 类型输入项
- **Then** 系统不打开空白侧边栏
- **Then** 当前主界面的设备选择、页签、日志列表保持不变

#### 场景：配置 AI 后打开侧边栏
- **Given** 用户已填写 API URL、API Key、模型、Provider 类型
- **When** 用户保存配置成功并再次点击 AI 图标
- **Then** 系统从右侧打开浮动侧边栏
- **Then** 侧边栏展示当前 Provider、模型、当前设备上下文
- **Then** 页面不会切换路由，不会重置当前页签

#### 场景：普通对话成功
- **Given** AI 已配置且 Provider 可用
- **When** 用户在侧边栏输入问题并发送
- **Then** 侧边栏展示用户消息和 AI 回复
- **Then** 回复过程中展示进行中状态
- **Then** 回复结束后输入框恢复可编辑

#### 场景：分析当前设备日志
- **Given** AI 已配置，当前选中设备为已连接 Android 或 iOS 设备
- **When** 用户点击“分析当前设备日志”或发送等价对话指令
- **Then** 系统以当前设备为分析对象获取日志上下文
- **Then** 日志上下文在提交 AI 前完成敏感信息脱敏和条数限制
- **Then** AI 回复包含问题摘要、关键证据日志、原因判断、建议操作、置信度
- **Then** 日志页已有日志和过滤条件不被清空或修改

#### 场景：用户已有手动日志采集
- **Given** 用户已在日志页手动启动当前设备实时日志采集
- **When** 用户在 AI 侧边栏请求分析当前设备日志
- **Then** AI 分析复用已有日志上下文
- **Then** 系统不得关闭用户手动启动的采集连接
- **Then** 系统不得重建导致日志页清空的采集连接

#### 场景：未连接设备请求日志分析
- **Given** 当前没有选中已连接设备，或当前设备状态不是已连接
- **When** 用户请求分析当前设备日志
- **Then** 系统不发起 AI 日志分析请求
- **Then** 侧边栏展示“当前没有可分析的已连接设备”
- **Then** 用户可继续普通 AI 对话

#### 场景：Provider 鉴权失败
- **Given** AI 已配置，但 API Key 无效
- **When** 用户发起普通对话或日志分析
- **Then** 侧边栏展示鉴权失败提示
- **Then** 提示内容不包含 API Key、Authorization Header、完整请求体
- **Then** 当前设备管理功能继续可用

#### 场景：关闭 AI 侧边栏
- **Given** AI 侧边栏已打开
- **When** 用户点击关闭按钮或按 Esc
- **Then** 侧边栏关闭
- **Then** 当前主界面的设备选择、页签、日志列表、文件选择、应用列表保持关闭前状态

#### 3. 失败处理

| 失败场景 | 处理方式 |
|---------|---------|
| 后端不可达 | 点击 AI 图标时展示后端未连接提示，不展示配置表单，不影响现有页面交互 |
| AI 未配置 | 打开配置弹窗，阻止进入空白 AI 侧边栏 |
| API URL 为空 | 配置表单阻止保存并提示 API URL 必填 |
| API Key 为空 | 配置表单阻止保存并提示 API Key 必填 |
| 模型为空 | 配置表单阻止保存并提示模型必填 |
| Provider 类型为空 | 配置表单阻止保存并提示 Provider 类型必填 |
| 连接测试失败 | 保留用户输入，展示脱敏错误，可重新测试或取消 |
| AI 请求超时 | 终止当前请求，展示超时提示，允许用户重试 |
| AI 返回格式不兼容 | 展示 Provider 响应不兼容提示，保留用户问题和上下文摘要 |
| 日志上下文为空 | 展示未获取到日志的提示，并给出启动日志采集或重新分析入口 |
| 日志上下文超过限制 | 截断上下文后继续分析，并在分析结果中展示已截断 |
| 当前设备断开 | 停止本次日志分析，展示设备已断开提示 |
| 用户取消回复 | 停止当前 AI 回复渲染，保留已生成内容，并允许继续提问 |

#### 4. 边界与约束

| 边界 | 说明 |
|------|------|
| 最小化主界面改动 | 主界面只允许新增 AI 入口挂载和必要的只读上下文传递；不得把 AI 对话、Provider、配置表单逻辑写入设备、文件、应用、日志页面主体逻辑 |
| 现有日志接口兼容 | 现有日志流、日志导出、日志过滤对外行为保持不变；新增 AI 能力不得改变原接口响应结构 |
| AI 配置安全 | API Key 不允许存入前端 localStorage、sessionStorage、URL、DOM 属性和错误提示；后端配置状态接口只返回脱敏信息 |
| 日志脱敏 | 发送给 AI 的日志正文必须脱敏 Authorization、token、手机号、邮箱、设备序列号、常见密钥字段 |
| Provider 基线 | 第一阶段以 OpenAI-compatible 文本对话为基线；Provider 特有能力不作为 P0 验收项 |
| 平台范围 | 日志分析 P0 覆盖 Android 和 iOS；HarmonyOS 在无实时日志能力时展示平台不支持日志分析 |
| 请求限制 | 单次日志分析必须限制日志条数、字符数和 AI 回复长度；限制值需在页面或返回信息中可见 |
| UI 层级 | AI 图标、配置弹窗、侧边栏层级高于主内容区，低于系统级错误弹窗；不得遮挡已有确认删除、卸载、重命名弹窗 |
| 构建约束 | 前端构建和后端测试通过；关闭或移除 AI 配置时现有功能仍可运行 |
| 可观测性 | AI 请求记录 Provider、模型、耗时、成功或失败状态；不得记录 API Key 和完整日志正文 |
