# AI 助手毛玻璃效果不可见排查记录

## 排查步骤

- [x] 复核用户症状：用户反馈未看到高斯模糊效果，视觉表现接近不透明。
- [x] 定位相关代码：检查 `DevBridge-Front/src/app/ai/AiChatPanel.tsx` 中 AI 侧栏容器、标题区、设备上下文和输入区的背景透明度。
- [x] 验证构建链路：确认 Tailwind 任意透明度类可被生产构建生成。
- [x] 修复样式参数：降低侧栏和分区背景不透明度，保留输入框和卡片较高可读性。
- [x] 重新构建并重启服务：生成 `V2026.7.0048`，确认 8080 首页引用新资源。

## 假设与验证

| 假设 | 验证结果 |
| --- | --- |
| `backdrop-blur` 没有生成 CSS | 不成立。构建产物包含 `backdrop-blur-2xl`，之前浏览器计算样式也显示 `blur(40px)`。 |
| 浏览器不支持高斯模糊 | 不成立。当前浏览器可计算并应用 `backdrop-filter`。 |
| 背景透明度过高导致肉眼看不出模糊 | 成立。主体 `bg-background/[0.92]`、头部/底部 `0.86` 接近实体背景，模糊效果被背景色覆盖。 |

## 根因定位链

用户看到“不透明” → 检查 AI 侧栏主体样式 → 发现已经有 `backdrop-blur-2xl` → 继续检查背景色 alpha → 主体为 0.92，头部/底部为 0.86 → alpha 过高导致下层内容几乎不可见 → 视觉上等同不透明，毛玻璃效果不可感知。

## 修复方案

- 将 AI 侧栏主体背景从 `0.92` 降到 `0.74`，暗色模式降到 `0.72`。
- 将标题区、工具区、设备上下文和输入区分层透明度分别调低，形成可见的玻璃层次。
- 输入框、用户消息、过程卡片等承载内容的局部区域保留更高不透明度，避免可读性下降。
- 在 JSX 中保留中文修复注释，说明为什么不能把玻璃容器设置得过于不透明。
