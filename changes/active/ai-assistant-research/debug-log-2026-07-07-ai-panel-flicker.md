# AI 助手处理中闪烁排查记录

## 排查步骤

- [x] 复核用户症状：AI 助手在发送消息和“正在处理”期间出现窗口闪烁。
- [x] 搜索 AI 面板内的动画、毛玻璃、加载状态和滚动更新代码。
- [x] 阅读 `AiChatPanel.tsx`，确认请求处理中没有定时切换 `loading` 状态，闪烁不是业务状态反复变更导致。
- [x] 对比近期全屏改动，确认全屏只扩大 fixed 容器范围，未改变消息流和请求状态逻辑。
- [x] 定位持续动画点：`正在处理`、过程卡片运行态、工具步骤运行态均使用 `Loader2 animate-spin`。
- [x] 修复后执行前端构建验证。

## 假设与验证

| 假设 | 置信度 | 验证结果 |
| --- | --- | --- |
| 请求状态反复 setLoading 导致闪烁 | 低 | `loading` 只在请求开始和结束切换，不会在处理中高频变化。 |
| 滚动逻辑导致闪烁 | 中 | `scrollIntoView` 只随消息、提示、输入高度和 loading 变化触发，不会按动画帧持续触发。 |
| 毛玻璃背景叠加旋转动画导致大面积合成重绘 | 高 | 面板使用 `backdrop-blur-2xl`，加载图标持续 transform 动画；全屏后 blur 覆盖面积变大，问题更容易出现。 |

## 根因链路

AI 助手面板为了毛玻璃效果使用了全尺寸 `backdrop-filter` 背景层。处理中状态内部存在多个 `animate-spin` 旋转图标。浏览器在处理 `backdrop-filter` 与子元素持续 transform 动画时，会频繁重算合成层；全屏后影响面积扩大，视觉上表现为整个 AI 助手窗口闪烁。

## 修复方案

将 AI 面板内持续旋转的加载图标替换为静态状态点：

- `正在处理` 状态使用静态主色状态点。
- 过程卡片运行态使用同一静态状态点。
- 工具步骤运行态使用同一静态状态点。

这样保留处理中语义，但移除每帧动画对毛玻璃合成层的刺激，避免整窗闪烁。修复不改变 AI 对话、MCP 工具调用、历史聊天、全屏和配置逻辑。
