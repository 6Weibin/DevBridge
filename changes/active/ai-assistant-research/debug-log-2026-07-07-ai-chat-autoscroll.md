# AI 助手聊天自动滚动排查记录

## 排查步骤

- [x] 复核症状：点击展开/折叠思考与执行过程卡片时，聊天区域自动滚动到底部。
- [x] 搜索 `AiChatPanel.tsx` 中滚动逻辑、过程卡片展开逻辑和消息状态更新逻辑。
- [x] 确认 `toggleProcess` 会通过 `setMessages` 修改 `messages`。
- [x] 确认旧滚动 effect 依赖 `messages`，任何消息结构变化都会触发 `scrollIntoView` 到底部。
- [x] 设计粘底模式：只有滚动区本身处于底部附近时才自动跟随新内容。
- [x] 增加用户上滑后的回到底部按钮。
- [x] 执行前端构建验证。

## 假设与验证

| 假设 | 置信度 | 验证结果 |
| --- | --- | --- |
| 展开/折叠过程卡片触发了消息变化 | 高 | `toggleProcess` 修改 `message.process.expanded`，会触发 `messages` 变更。 |
| 自动滚动没有区分新内容和查看历史 | 高 | 旧 effect 依赖 `messages`，只要消息数组变化就无条件滚到底部。 |
| 用户上滑后仍被强制拉回底部 | 高 | 旧逻辑没有记录用户当前是否在底部，也没有暂停自动跟随机制。 |

## 根因链路

过程卡片的展开/折叠属于“查看历史内容”的 UI 状态变化，但实现上复用了 `messages` 状态。旧自动滚动 effect 把所有 `messages` 变化都当作“新内容到达”，无条件执行 `scrollIntoView`，因此用户点击展开/折叠时会被强制拉到底部。

## 修复方案

- 为聊天内容区增加滚动容器 `ref` 和底部状态判断。
- 增加“粘底模式”：只有用户当前在底部附近时，新消息、流式回复、处理中状态变化才自动跟随到底部。
- 用户主动上滑后自动关闭粘底模式。
- 点击展开/折叠思考过程时设置一次跳过自动滚动标记。
- 非底部状态下显示圆形“回到底部”按钮，用户点击后恢复粘底模式。
- 新建、切换、删除会话时默认回到底部，避免继承上一会话的历史查看滚动状态。
