# Spec：跨平台 USB 手机设备管理工具 MVP

> 来源：跳级模式，基于原始需求文档、技术预研、PoC 验证结果和当前代码现状  
> 生成时间：2026-06-29  
> 阶段：spec

### Why

当前 PoC 已完成 Spring Boot 本机服务、React H5 Demo 接入、内置 macOS/Windows ADB、工具状态探测、设备枚举、运行环境诊断和基础演示日志。PoC 证明 Java + H5 + 本机 CLI 适配层可行，且 Android 真机枚举链路已通过项目内置 ADB 验证。

下一阶段目标是把 PoC 推进为可验收的 MVP：用户在 macOS 或 Windows 本机连接 Android 手机后，能在 H5 页面看到真实设备、读取基础设备信息、浏览公共目录、下载文件、查看实时 logcat、导出日志；HarmonyOS 和 iOS 在工具可用时提供设备发现和日志入口，在工具缺失或能力受限时给出明确状态。

### What

| 功能 | 验收标准 | 优先级 |
|------|---------|--------|
| 平台与工具状态诊断 | 前端显示后端运行平台目录；`/api/tools/status` 返回 adb、hdc、idevice 工具的可用状态、路径、版本摘要和缺失原因；工具缺失不影响页面打开 | P0 |
| Android 设备发现 | 连接已授权 Android 手机后，`/api/devices` 返回设备 serial、platform、status、model；未授权设备返回 `unauthorized`；离线设备返回 `offline`；无设备返回空列表 | P0 |
| Android 设备详情 | 选择已连接 Android 设备后，页面展示品牌、型号、Android 版本、API 级别、电量、屏幕分辨率和存储摘要；读取失败的字段显示 `—` 并保留设备基础信息 | P0 |
| Android 公共目录浏览 | 用户可浏览 `/sdcard`、`/storage/emulated/0` 下的目录和文件；目录列表包含名称、类型、大小、修改时间和权限摘要；越界路径请求被拒绝 | P0 |
| Android 文件下载 | 用户从允许目录选择单个文件后可下载到浏览器；下载失败时页面显示失败原因；取消或失败后服务端清理临时文件 | P0 |
| Android 实时日志 | 用户可启动、暂停、停止 Android logcat 实时日志；日志包含时间、级别、tag、pid、message；前端刷新或断开后后端停止对应日志进程 | P0 |
| Android 日志导出 | 用户可按当前设备导出 logcat 快照；导出完成后浏览器下载日志文件；导出失败时返回明确错误 | P0 |
| HarmonyOS 设备发现降级 | hdc 可用时返回 HarmonyOS 设备 serial 和授权状态；hdc 缺失时工具状态显示缺失原因，HarmonyOS 功能入口置为不可用 | P1 |
| iOS 设备发现与日志降级 | idevice 工具可用时返回 iOS UDID 和连接状态，并提供系统日志入口；Windows 或工具缺失时显示依赖缺失说明；不提供 iOS 通用文件树 | P1 |
| 前端真实数据接入 | 设备列表、工具状态、设备详情、文件树、日志窗口优先使用后端真实数据；后端不可达时展示演示数据并显示降级提示 | P0 |
| 本机安全边界 | 后端只监听 `127.0.0.1`；所有文件访问限定在设备允许目录和应用临时目录；用户输入不能触发 shell 拼接；错误响应不泄露本机敏感路径之外的完整命令输出 | P0 |

**不做的事：**  
- 不实现 root、越狱、MDM、云端设备农场、远程代理和驱动自动安装。  
- 不实现 iOS 未越狱设备的通用文件系统浏览。  
- 不实现 HarmonyOS hdc 官方包自动下载。  
- 不实现桌面安装包、自动升级和托盘常驻。  
- 不实现多用户权限系统。

### How

#### 1. 正常流程

1. 用户启动后端服务并打开 H5 页面。
2. 页面加载运行环境和工具状态，显示当前平台目录、adb/hdc/idevice 可用状态。
3. 用户通过 USB 连接 Android 手机并完成 USB 调试授权。
4. 用户点击刷新，页面展示真实 Android 设备。
5. 用户选择 Android 设备，页面加载设备详情。
6. 用户进入文件页，浏览允许目录下的文件和目录。
7. 用户选择文件下载，浏览器收到文件。
8. 用户进入日志页，启动实时日志流，页面持续追加日志行。
9. 用户停止日志或关闭页面，后端停止对应日志进程。
10. 用户点击导出日志，浏览器下载日志文件。

#### 2. 关键场景

#### 场景：已授权 Android 设备被发现
- **Given** 后端运行在本机，项目内置 adb 可用，Android 手机已开启 USB 调试并授权
- **When** 用户请求设备列表
- **Then** 系统返回一条 platform 为 `android`、status 为 `connected`、serial 非空的设备记录

#### 场景：Android 设备未授权
- **Given** Android 手机已连接 USB，手机端未确认 RSA 授权
- **When** 用户请求设备列表
- **Then** 系统返回该设备 status 为 `unauthorized`，前端显示“待授权”状态

#### 场景：浏览 Android 公共目录
- **Given** Android 设备处于 `connected` 状态
- **When** 用户请求 `/sdcard/Download` 目录
- **Then** 系统返回该目录下的文件和目录列表，列表字段包含 name、path、type、size、modified

#### 场景：拒绝越界路径
- **Given** Android 设备处于 `connected` 状态
- **When** 用户请求包含 `..` 或不在允许根目录下的远端路径
- **Then** 系统拒绝请求并返回业务错误，设备命令不执行

#### 场景：实时日志断开清理
- **Given** 用户已启动 Android 实时日志
- **When** 前端主动停止、刷新页面或连接断开
- **Then** 后端结束对应日志进程，并释放该日志会话资源

#### 场景：后端不可达时前端降级
- **Given** H5 页面已打开，后端服务未启动
- **When** 页面刷新数据
- **Then** 页面展示演示数据，并显示“未连接本机后端，当前展示演示数据”

#### 3. 失败处理

| 失败场景 | 处理方式 |
|---------|---------|
| adb 不存在 | `/api/tools/status` 返回 adb unavailable，前端 Android 功能入口置为不可用 |
| adb 命令超时 | 后端终止命令，接口返回超时错误，前端显示“命令超时” |
| 设备未授权 | 设备状态返回 `unauthorized`，前端提示用户在手机上允许 USB 调试 |
| 设备离线 | 设备状态返回 `offline`，文件和日志入口不可执行 |
| 目录不存在 | 文件列表接口返回路径不存在错误，前端保持当前目录 |
| 文件下载中设备断开 | 下载任务失败，前端显示失败原因，后端清理临时文件 |
| 日志进程异常退出 | 日志流结束并推送错误事件，前端停止播放状态 |
| hdc 缺失 | HarmonyOS 工具状态显示 `tool not found`，HarmonyOS 操作入口置为不可用 |
| idevice 工具缺失 | iOS 工具状态显示 `tool not found`，iOS 操作入口置为不可用 |

#### 4. 边界与约束

| 边界 | 说明 |
|------|------|
| 本机监听 | 后端服务仅监听 `127.0.0.1` |
| Android 允许目录 | 文件浏览和下载仅允许 `/sdcard`、`/storage/emulated/0` |
| 单文件下载 | MVP 仅支持单文件下载，不支持目录打包下载 |
| 下载临时文件 | 服务端下载临时文件必须在任务完成、取消或失败后清理 |
| 日志行缓存 | 前端单个日志视图最多保留 1000 行，超过后丢弃最旧日志 |
| 日志会话并发 | 单个设备同一时间最多 1 个实时日志会话 |
| iOS 文件能力 | MVP 不显示 iOS 通用文件树 |
| HarmonyOS 文件能力 | MVP 不承诺 HarmonyOS 文件下载，只做设备发现降级 |
| 工具来源 | adb 使用项目内置工具优先；hdc 和 idevice 使用配置路径或 PATH |
| 错误输出 | API 错误返回错误码和摘要，不返回完整敏感日志内容 |
