# 文件夹空目录和无权限顶部提示调整记录

## 排查步骤

- [x] 检查空目录和不可访问目录提示写入位置。
- [x] 确认旧逻辑会把提示写入文件树顶部 `fileHint` 或顶栏 `actionHint`。
- [x] 新增独立顶部浮层提示状态，避免占用文件树路径下方的小字状态区。
- [x] 调整空目录、无权限/不可访问目录的打开反馈，只通过顶部浮层提示。
- [x] 执行前端构建、后端打包和服务重启验证。

## 根因

目录状态提示复用了文件管理顶部的小字状态区和全局动作提示区。该区域更适合展示当前路径加载状态，不适合作为空目录、无权限这类瞬时反馈。

## 修复方案

- 新增 `topTip` 和 `showTopTip`。
- 顶部居中显示短时浮层提示，自动消失。
- 空目录加载成功后：`fileHint` 只显示 `已加载 path`，空目录信息改由顶部浮层提示。
- 无权限/不可访问目录加载失败后：不再覆盖 `fileHint`，改由顶部浮层提示。
- 双击已知空目录或不可访问目录时，只触发顶部浮层提示。
