# 文件夹空目录和无权限状态加载时标记记录

## 排查步骤

- [x] 复查现有目录状态逻辑，确认目录变浅依赖用户双击打开后记录状态。
- [x] 检查后端文件列表接口，确认当前目录列表只返回直接子节点，不包含子目录内容数量。
- [x] 在前端当前目录加载完成后，对直接子目录做受控预探测。
- [x] 将预探测结果写入设备维度目录状态缓存，供文件树立即渲染浅色目录名称。
- [x] 执行前端构建、后端打包和服务重启验证。

## 根因

上一次实现只在 `openFileDirectory` 中根据打开结果记录 `empty` 或 `inaccessible` 状态，因此必须点击或双击目录后才会变浅。

用户期望是在当前目录列表加载完成时就能看到哪些目录为空或不可访问。由于后端 `ls -la` 当前目录只能返回子目录本身的元信息，不能直接知道子目录内容是否为空，所以前端需要在加载当前目录后主动探测当前列表中的目录。

## 修复方案

- 新增 `preloadDirectoryStates(nodes)`：
  - 当前目录加载成功后筛选其中的目录节点。
  - 按 4 个一批并发请求子目录列表，避免一次性发起过多请求。
  - 子目录返回空列表时记录为 `empty`。
  - 子目录接口失败时记录为 `inaccessible`。
- `loadFiles` 成功后立即触发预探测，不再依赖目录双击。
- 文件树继续复用 `directoryStates` 渲染浅色名称和浅色目录图标。
- 双击已知空目录或不可访问目录时继续给出顶部提示和文件区提示。
